  <action>
- Create `PromptProfiles.kt` with Phase 4 models (keep in `com.github.shekohex.whisperpp.data`):
  - `PromptProfile(id, name, promptAppend)`.
  - `AppPromptMapping(packageName, profileId?, appendMode, appPromptAppend?, sttOverride?, textOverride?)`.
  - `AppendMode` MUST support: append (default) and `NO_APPEND` (locked: disable append while still keeping global base prompt).
  - Provider/model override type MUST allow partial fields (providerId and/or modelId can be blank) to support locked cascading.
  - Add a warning/issue model (e.g., `RuntimeWarning`) suitable for non-blocking UI notices.

- Create `PromptComposer.kt` implementing locked prompt composition:
  - Inputs: `basePrompt`, optional `profileAppend`, optional `appAppend`, and `appendMode`.
  - Output: one effective prompt string.
  - MUST preserve order exactly: base + profile append + app-specific append.
  - If `appendMode == NO_APPEND`: ignore profile and app appends, but still return base.
  - Omit blank segments (no trailing blank blocks).

- Create `RuntimeSelectionResolver.kt`:
  - Implement one entrypoint (e.g., `resolve(...)`) used later by BOTH dictation (STT) and enhancement (TEXT).
  - Locked precedence for provider/model:
    - `app override > per-language default > global default`.
    - Partial overrides cascade missing fields from per-language, then global.
    - Resolve STT and TEXT channels independently.
  - Validation rules:
    - If the resolved pair references a missing provider or missing model under provider, fallback down precedence (language then global), and emit a non-blocking warning.
    - If mapping references a deleted/missing profile ID: fallback to global-only prompt behavior and emit a warning.
    - (Reasonable discretion) validate model kind compatibility:
      - STT channel requires `ModelKind.STT` or `ModelKind.MULTIMODAL`.
      - TEXT channel requires `ModelKind.TEXT` or `ModelKind.MULTIMODAL`.
      - On mismatch: treat as invalid and fallback + warning.
  - Return a single `EffectiveRuntimeConfig` containing:
    - Effective STT providerId/modelId
    - Effective TEXT providerId/modelId
    - Effective prompt string (from PromptComposer)
    - `appendMode` used and any warnings
  - Do NOT read DataStore here; accept plain inputs (packageName, languageCode, providers list, language defaults list, app mappings list, global default IDs, base prompt string, profiles list).
