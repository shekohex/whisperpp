  - provider credentials
  - active selections
  - language defaults
  - prompts/profiles
  - app mappings
  - transform presets
  - keyboard behavior
  - privacy/safety
  - advanced preferences

- Merge semantics must be explicit and stable:
  - list-by-id categories (`providers`, `promptProfiles`) upsert by ID with imported item winning
  - list/map-by-package categories (`appMappings`, send-policy rules) overwrite conflicting package entries with imported data
  - scalar/toggle categories overwrite the local value when the category is included
  - provider credentials restore back into `SecretsStore`, never into provider JSON

- After apply, run post-restore validation before returning success:
  - call existing `validateSelections()` and clear invalid provider/model keys
  - detect providers that require API keys (`authMode == API_KEY`) but have missing/unreadable restored credentials
  - if missing credentials make active selections unusable, clear those selections and include actionable repair entries pointing to the affected provider(s) and selection area(s)

- Return a typed restore summary the UI can render inline after apply. It must include:
  - applied categories
  - skipped items
  - cross-version warnings
  - selections cleared
  - repair checklist entries for providers/selections needing attention
