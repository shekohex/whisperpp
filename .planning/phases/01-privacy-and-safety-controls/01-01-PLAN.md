  <files>
android/app/src/main/java/com/github/shekohex/whisperpp/WhisperTranscriber.kt
  </files>
  <action>
- Track the active OkHttp Call:
  - Add a `private var inFlightCall: Call?`.
  - In `makeWhisperRequest()`, create the call via `client.newCall(request)` and assign to `inFlightCall` before `execute()`.
  - Ensure `inFlightCall` is cleared (set null) on all exit paths (success, error, cancellation).

- Ensure `stop()` cancels real network work:
  - In `stop()` (or `registerTranscriptionJob(null)`), call `inFlightCall?.cancel()` then clear it.
  - Keep coroutine job cancellation as-is, but do not rely on it for network cancellation.

- Treat cancel-triggered IO as benign cancellation:
  - When `execute()` throws `IOException`, if `inFlightCall?.isCanceled() == true` OR the coroutine context is cancelled, treat it as cancellation.
  - Do NOT call `exceptionCallback(...)` and do NOT surface a user-facing error/toast for this case.
  - Still clear `inFlightCall`.

- Remove any error propagation that includes raw response bodies:
  - When `!response.isSuccessful`, do NOT read/return/log `response.body.string()`.
  - Instead, derive a safe error string including only: provider name (if available in scope), HTTP status code, and endpoint host (domain only).
    - Parse host using OkHttp `HttpUrl` (e.g., `provider.endpoint.toHttpUrlOrNull()?.host`). Do not show full URL path/query.
  - Ensure `exceptionCallback(...)` only receives safe text.

- Fix response body consumption:
  - Read response body string at most once.
  - Do not call `response.body!!.string()` twice.

- Logging:
  - Default must not log request/response bodies.
  - Keep `Authorization` redacted if headers logging is enabled.
  - Introduce a simple runtime switch (e.g., `setNetworkLoggingEnabled(enabled: Boolean)`) that toggles `HttpLoggingInterceptor` between `NONE` (default) and `HEADERS` (verbose), never `BODY`.
