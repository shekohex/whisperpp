  <action>
- Provider model:
  - In `ServiceProvider`, mark `apiKey` as non-serializable for Gson (e.g., `@Transient val apiKey: String = ""`).
  - Keep constructor compatibility so existing call sites compile.

- Settings export/import:
  - Ensure exported `providers` never contain API keys.
  - Remove secret keys from exportable preference key lists (e.g., `"api-key"`, `"smart-fix-api-key"`).
  - Import must ignore any secret keys if they exist in an imported file.

- Plaintext migration (must avoid losing existing users’ keys):
  - Add a one-time migration guard flag (boolean) in DataStore, e.g. `PROVIDER_API_KEY_MIGRATION_DONE`.
  - Add a one-time migration function that MUST NOT rely on `ServiceProvider` (since `apiKey` becomes non-serializable):
    1) If `PROVIDER_API_KEY_MIGRATION_DONE == true`, return immediately.
    2) Read the raw `providers_json` string exactly as stored (do not deserialize into `ServiceProvider`).
    3) Parse as JSON using a tree API (Gson `JsonParser` / `JsonElement`):
       - Support top-level `JsonArray` of provider objects.
       - Also support top-level `JsonObject` that contains an array field (e.g., `providers`), if your existing persisted shape uses that.
       - If parsing fails or shape is unknown: do not modify anything and do not set the flag.
    4) For each provider JSON object:
       - Extract provider id directly from JSON (use the persisted id field used today, typically `id`). If missing/blank, skip.
       - Extract `apiKey` directly from JSON (field name `apiKey`). If present and non-blank:
         - Store into `SecretsStore.setProviderApiKey(providerId, apiKey)`.
       - Remove the `apiKey` property from the JSON object (do not blank it; remove it).
    5) Write the sanitized JSON back to `providers_json` (same shape as input).
    6) Set `PROVIDER_API_KEY_MIGRATION_DONE=true` only after successfully writing sanitized JSON.
  - Ordering constraints (to prevent key loss):
    - Call migration BEFORE any code path that reads/deserializes providers into `ServiceProvider`.
    - Invoke this migration early in both Settings app and IME service paths:
      - from Settings UI entry (e.g., in `SettingsNavigation` via a `LaunchedEffect`)
      - from `WhisperInputService.onCreate()`
  - Migration must not log extracted keys and must never surface them in exceptions.

- Provider edit UI:
  - Never prefill API key field from model.
  - Show “Key set” state + last4 from `SecretsStore`.
  - On Save:
    - persist provider config without apiKey
    - if user entered a non-empty key, write it to `SecretsStore`
  - Provide Clear/Replace key actions; no copy/export.

- Runtime usage:
  - Before calling `WhisperTranscriber.startAsync(...)` or `smartFixer.fix(...)`, fetch api key from `SecretsStore` for the chosen provider and pass it by copying provider (`provider.copy(apiKey = key)`), without persisting it.
  </action>
