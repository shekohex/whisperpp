  </files>
  <action>
- Create package `com.github.shekohex.whisperpp.command`.

- Add `CommandModels.kt` containing small, testable models:
  - `data class SelectionSnapshot(start: Int, end: Int)`
  - `sealed class ResolvedSelection`:
    - `Selected(text: String, snapshot: SelectionSnapshot)`
    - `None(reason: Reason)` where Reason is an enum (e.g., `NO_SELECTION`, `TYPE_NULL`, `UNKNOWN`).
  - `data class CommandUndoEntry(focusKey: FocusKey, snapshot: SelectionSnapshot, originalText: String, appliedText: String, createdAtMs: Long, expiresAtMs: Long)`

- Implement `SelectionResolver.kt`:
  - Provide a best-effort API:
    - Uses `InputConnection.getExtractedText(ExtractedTextRequest(), 0)` to obtain `selectionStart/End`.
    - Treats invalid indices (<0) as None.
    - Treats start==end as None.
    - Uses `InputConnection.getSelectedText(0)` as best-effort for text; if null/blank, still treat as None.
    - If `EditorInfo.inputType == 0` (TYPE_NULL), treat as None (force clipboard fallback).

- Implement `CommandController.kt` as a small state holder (not doing I/O):
  - Owns a `runId` (increment per start) and current `FocusKey`.
  - Stores current `CommandUndoEntry?` and exposes helpers:
    - `fun clearUndo(reason: String)` (called by service on next run)
    - `fun shouldShowUndo(nowMs: Long): Boolean` (false when expired)
  - Adds a helper to create `CommandUndoEntry` after apply succeeds (service provides original/applied text + selection snapshot + focusKey + nowMs).
  - Enforces expiry: 30 seconds from creation.

- Unit tests:
  - `SelectionResolverTest` covering: no selection, selection present, TYPE_NULL forces None.
  - `CommandControllerTest` covering: undo expiry at 30s boundary and cleared on next-run.
  </action>

    <automated>./android/gradlew -p android testDebugUnitTest</automated>
  </verify>
  <done>
- Selection resolution and command undo contract exist, are deterministic, and covered by unit tests.
  </done>
</task>
