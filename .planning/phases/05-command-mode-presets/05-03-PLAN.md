  - On enter, capture `FocusKey` for the current editor.
  - Enforce external-send gate before any recording/transcribing/LLM action:
    - If `shouldBlockExternalSend()` is true, do not enter command mode; route to existing blocked explanation flow (reuse existing mechanism).

- Resolve text source (CMD-02):
  - Use `SelectionResolver` on `currentInputConnection` + `currentInputEditorInfo`.
  - If selection exists:
    - Store selection snapshot + original selected text for undo.
    - Proceed to instruction listening.
  - If no selection:
    - Enter clipboard fallback stage and require explicit confirmation (locked decisions):
      - Show a preview snippet + full character count.
      - Require a user tap on Continue before using clipboard text.
      - If clipboard text is very large, show a warning but allow continue (choose threshold; show actual count).
      - Allow two attempts total for clipboard fallback (initial + one retry); then abort with guidance and exit command mode with no text changes.
      - If clipboard access unavailable or empty, show reason + manual select/copy recovery path.
      - Do not reuse previously confirmed clipboard text across separate command runs; each new run must re-confirm.

- Instruction capture (CMD-03):
  - After text is confirmed (selection or clipboard), start instruction recording.
  - Support cancel at any time and Command-key toggle to exit (locked).
  - Transcribe instruction using existing RecorderManager + WhisperTranscriber path (non-streaming, short clip):
    - Use effective STT provider/model already used for dictation (via existing resolver wiring).
    - If transcript is blank/silence: reprompt once; if still blank, exit command mode cleanly.
  - Enforce `shouldBlockExternalSend()` before STT start and before LLM call (single shared gate rule).
  - Ensure cancellation is benign and cancels underlying OkHttp Call (existing decision).

- LLM transform call (CMD-03):
  - Resolve effective TEXT provider/model for command mode:
    - Use RuntimeSelectionResolver command inheritance behavior (command override keys may be unset to inherit enhancement selection).
    - If missing selection, show setup-needed style error and exit (no text changes).
  - Gate through first-use disclosure for command text mode (existing mode-specific disclosure plumbing).
  - Build prompt using:
    - Effective composed base prompt (existing PromptComposer output)
    - Selected preset instruction (from TransformPresets + current chosen preset ID)
    - Spoken instruction transcript
    - Confirmed text (selection or clipboard)
    - MUST include explicit “spoken overrides preset on conflict” rule (locked).
  - Call SmartFixer (or existing text transform path) with the effective provider/model.
  - On timeout/failure: keep original text unchanged and show error with Retry CTA (locked).
    - Retry should rerun the LLM call using the same confirmed text + same spoken instruction within the same command run.

- Apply + undo (CMD-04):
  - Apply replacement by immediate replacement of the selected text:
    - Use batch edit.
    - Ensure selection is set to the original range then replaced via commitText.
    - After apply, keep resulting text selected (locked).
  - Create a command undo entry and expose 1-tap undo:
    - Undo restores original text exactly.
    - Undo is available until next command run OR 30 seconds, whichever comes first (locked).
    - After apply or undo, keep resulting/restored text selected (locked).
  - On cancel at any stage (including fallback failure): exit command mode with no text changes (locked).
